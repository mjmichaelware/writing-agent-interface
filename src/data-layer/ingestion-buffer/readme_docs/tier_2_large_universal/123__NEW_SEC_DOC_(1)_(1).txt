### Phase 2: Runtime Model Shift (Middle session)
- Moved from "UI design" thinking в†’ "deterministic state machine"
- Initially proposed plugin system; user corrected to event-driven (simpler, proven pattern)

- Built v1 scaffold (clean structure, dead code)


### Phase 3: Operational Reality Check (Mid-late session)
- v1 had 6 real bugs: listeners never initialized, block:render never emitted, retreat() bypassed engine, no cleanup, coupling risks
- User pushed back: "not philosophy, actual risk analysis"
- Built v2 patch: singleton pattern, command enforcement, proper emissions
- Deployed to Vercel (build failed due to missing files)


### Phase 4: Smart Parser Recognition (Most recent)
- User rejected hardcoded chapter data
- System should intelligently: detect device в†’ calculate words-per-screen в†’ fetch raw chapter в†’ parse responsive blocks
